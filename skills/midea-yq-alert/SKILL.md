# 美的舆情预警报送判定 Skill

版本：V5.1.0
更新日期：2026-06-16

## 定位与架构角色

本 Skill 是**数据采集系统 → 客户推送**之间的**智能中台判定层**。

```
数据采集系统(基础标签+召回) → 本Skill(是否报送+格式化) → 客户推送(标准5段式)
                      ↑                                    ↓
               输入: 标题/摘要/正文/              输出: 报送/不报送/
               链接/平台/发布人/                   推送标题/摘要/
               系统标签/情感等                    AI总结/AI研判
```

**数据系统已完成的**：内容相关性初筛、基础标签打标（如美的业务部1/2、内容标签AI、负面等级AI）、关键词召回。

**本 Skill 负责的**：基于语义判断决定"报/不报"，并按客户要求的**报送格式**输出完整标签。

## 任务目标

对 AI 监测系统输出的舆情数据逐条执行：

1. **报送判定**：该条信息美的需不需要报送？
2. **格式转换**：若需报送，将系统基础标签转换为**客户报送格式**（严格按录表顺序）。
3. **去重**：识别已报送/重复/传播链，不重复推送。
4. **内容整理**：重写摘要/AI总结/AI研判，让客户不打开链接也能看懂。

> **关键概念：填表 vs 报送是两套体系**
>
> - **填表（录入系统）**：使用原始一级标签，如【创始人及高管舆情】作为一级标签
> - **报送（推给客户）**：客户指定了新的录表顺序/显示格式，一级标签位置可能为空或使用不同映射
> - 本 Skill 的输出统一为**报送格式**

## V5.0 正式标准优先级

本 skill 以《美的项目舆情打标判定手册_正式版V5.0》和《美的项目高频打标案例库_正式版V5.0》为最高优先级标准。

统一执行八步流程：

1. 是否属于美的监测范围
2. 是否需要报送
3. 主体判定
4. 分类判定
5. 危机等级判定
6. 高敏判定
7. 报送群判定
8. 输出报送内容

禁止跳步，禁止只因单个关键词或单个 AI 标签直接得出最终结论。

## 第一部分：报送输出格式规范（最高优先级）

### 客户报送格式（录表顺序）

所有报送输出必须严格遵循以下 **5 段式标签格式**：

```
【{等级}】【{一级标签}】【{事业部}】【{业务线}】【{舆情分类}】
```

高敏时在最前方追加，必须使用带感叹号的固定写法 `【高敏❗】`：

```
【高敏❗】【{等级}】【{一级标签}】【{事业部}】【{业务线}】【{舆情分类}】
```

#### 格式对照表（录表顺序）

| 字段 | 说明 | 可选值 |
|------|------|--------|
| **等级** | 危机等级 | 【异常事件】、【危机】、【重大危机】 |
| **一级标签** | 内容分类顶层 | 见下方"一级标签池" |
| **事业部** | 组织归属 | 见下方"事业部池" |
| **业务线** | 具体业务/产品线 | 见下方"业务线程池" |
| **舆情分类** | 细分类型 | 见下方"舆情分类池" |

#### ⭐ 核心报送规则：高管相关 vs 非高管集团信息

这是客户最新指定的**报送格式核心差异点**：

**规则 A — 高管/创始人相关信息的报送格式：**

当舆情涉及何享健、方洪波、王建国、顾炎民、管金伟等创始人及高管时，报送标签必须为：

```
【异常事件】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】
```

即：
- **等级**：按实际判定（异常事件/危机/重大危机）
- **一级标签/二级标签**：`集团层面`
- **事业部/三级标签**：`品牌整体`
- **业务线/四级标签**：`创始人及高管舆情`
- **舆情分类/五级标签**：`高管名誉`
- 这是 Word 标签规则中的固定五段标签，不允许拆成 `创始人及高管舆情` 或 `高管名誉` 二选一。

实际输出示例：
- `【异常事件】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】`
- `【危机】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】`
- `【高敏❗】【危机】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】`

**规则 B — 与高管无关的集团层面信息：**

当舆情属于集团层面但**不涉及**高管个人时，报送标签为：

```
【异常事件】【集团层面】【品牌整体】【{对应舆情分类}】
```

即：
- **等级**：按实际判定
- **集团非高管信息是 Word 规则中唯一允许四段输出的场景**，不要额外输出空标签位。
- **二级标签**：`集团层面`
- **三级标签**：`品牌整体`
- **四级标签**：根据实际内容选择——`经营战略` / `投资并购` / `资本运作` / `人资纠纷` / `合规诉讼` / `环保安全` / `产业生态` / `ESG` / `品牌形象` 等

实际输出示例：
- `【异常事件】【集团层面】【品牌整体】【经营战略】`
- `【危机】【集团层面】【品牌整体】【资本运作】`
- `【异常事件】【集团层面】【品牌整体】【合规诉讼】`

**规则 C — 非集团层面的常规报送格式（C端/B端）：**

对于智能家居事业群（C端产品/中国区域）和B端业务的常规舆情：

```
【{等级}】【{一级标签}】【{事业部}】【{业务线}】【{舆情分类}】
```

此时**一级标签不为空**，填写实际的一级标签值。

实际输出示例：
- `【异常事件】【C端产品风险舆情】【智能家居事业群】【家用空调】【产品质量】`
- `【危机】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【服务售后】`
- `【异常事件】【B端业务风险舆情】【工业技术事业部】【暖通家电部件】【产业生态】`

### 正文字段 news_content 使用要求

`news_content` 是本 Skill 的**最关键分析维度**：

- **必须用于**：主体确认、风险落点确认、反误报、反漏报、摘要重写、去重签名
- **不得用于**：直接输出给客户（只能通过摘要间接呈现）
- 若 `news_content` 为空 → 标记"正文缺失，仅基于标题摘要判定"
- 若 `news_content` 与摘要矛盾 → 以完整语义为准并保守表达

## 第三部分：八步判定流程

### 第1步：主体相关性确认

只有满足以下任一条件才进入后续判定：

1. 原文明确讨论美的集团或美的体系内的品牌/产品/服务/业务线/门店/渠道/官方账号/关联公司/管理层/产业板块
2. 原文未直接写出集团名，但链接/账号/投诉对象/店铺名/商品名/图片OCR/系统召回字段能证明涉事对象属于美的体系
3. 原文讨论竞品/行业/第三方，但美的是被比较/被评价/被影射/被追责一方
4. 原文讨论 ToB 业务或关联产业板块，且能证明属美的集团客户关注范围

**判为不相关（默认不报送）**：
- "美的"仅作普通语言（完美的、甜美的、优美的等）
- 人名/地名/店名/品牌名相似但语境不能证明属美的
- 非美的品牌/店铺/产品投诉仅因系统召回进入
- 文章只罗列多个企业/历史案例/行业背景，美的不是被评价/批评对象
- 海外业务中性/正面且不牵连国内消费者/服务标准/国内外区别对待/集团声誉

**主体不确定 → "需人工复核"**

#### V5.0 主体判定优先级

```
高管 > 具体产品业务线 > B端业务 > 中国区域 > 集团层面
```

- 涉及方洪波/何享健/创始人/高管任免/薪酬/名誉/传闻 → 优先归**高管/集团层面**
- 能识别具体产品线 → 归具体**产品业务线**（空调→家用空调，不归中国区域）
- 只能识别为“美的产品/美的家电/美的白电/买美的的”等泛产品表达，但无法确认具体产品线 → 仍属于**产品层面**，归 `【C端产品风险舆情】【智能家居事业群】【中国区域】`，不得归集团品牌整体
- 不能识别具体产品，但投诉客服/门店/安装/售后体系 → 归**中国区域**
- 库卡/安得/美云/楼宇科技/工业技术/万东 → 归**B端或其他业务**
- 只讨论集团战略/投资并购/资本市场/年报利润/年度经营/财报表现/战略方向/组织治理/高管/品牌整体声誉，且负面不落在具体产品或泛产品体验上 → 归**集团层面**

#### 主体性三问法

1. 这条内容主要在讲谁或什么对象？
2. 美的是被讨论主体/被投诉对象/被比较对象/被影射对象，还是仅被顺带提及？
3. 负面/敏感含义是否明确落到美的/关联品牌/产品/服务/渠道/高管/业务板块上？

**第2、3问题都答"是"才进入下一步。否则不报送。**

### 第2步：重复识别

**去重键优先级**：
1. 投诉编号键（黑猫/消费保/12315投诉编号）
2. 平台内容 ID 键（从链接提取唯一ID）
3. 规范化链接键（保留平台ID参数，不删nid/aid/mid/sn/item）
4. 内容签名键（平台+发布人ID+标题前80字+摘要前200字+正文前300字）
5. 传播链键（同一原文/视频/话题的转发链核心文本）

**各平台ID提取规则**：
- 黑猫：`/complaint/view/{编号}`
- 12315：`regiNo=` 或文本中的投诉编号
- 消费保："投诉编号"
- 抖音：`/video/{视频ID}`
- 小红书：`/explore/{笔记ID}` or `/discovery/item/{笔记ID}`（注意短链xhslink.com转换）
- 微博：微博正文ID/转发链原短链/正文核心文本
- 今日头条：`/item/{ID}` or `/w/{ID}`
- 微信文章：`__biz + mid + idx + sn`
- 视频号/快手/B站：视频ID

**重复处理**：
- 同一键已报送 + 无新增事实 → 不重复报送，输出"重复：已报送"
- 同一投诉状态流转（审核→分配→回复→完成），核心事实不变 → 不重复，提示"同投诉新进度"
- 同一投诉有新增事实/金额扩大/诉求升级/媒体介入 → 可报送为"进展更新"，标注"同投诉编号曾报送"
- 同一话题多平台扩散 → 首发/重点平台逐条报送，低互动标注"同话题传播链"

### 第3步：报送/不报送判定

#### 报送条件（满足任一即报）

1. **美的消费者风险**：用户/消费者/员工/媒体描述的具体问题落在美的关联产品/官方渠道/门店/物流/安装/售后/维修/价格权益/活动承诺上
2. **公开投诉风险**：公开投诉/监管受理/消费维权/媒体帮办中，投诉对象或核心事实指向美的关联主体
3. **美的不利对比**：竞品/评测/导购形成对美的不利结论，美的被质疑/贬低/认为存在短板
4. **集团/资本/高管敏感**：讨论美的集团/控股资产/高管/股东利益/资本承诺/经营压力/监管/诉讼/资产处置/亏损/名誉争议
5. **高敏安全/合规风险**：人身安全/公共安全/监管合规/环保/劳动关系/群体性投诉/社会敏感议题，风险落在美的
6. **传播链负面**：转发/评论/截图/表情包本身无新增事实，但核心内容对美的不利
7. **舆论高敏议题**：国内外差异/区别对待/弱势群体受损/公共安全/法律节目跟踪报道
8. **股民/资本情绪**：证券社区/财经媒体中负面落在美的集团/科陆/万东/能源等关联资产，且讨论已超出普通股价涨跌，指向经营问题、战略投资、业务布局、财报压力、资本承诺、品牌声誉或投资者集中质疑
9. **人资/用工风险**：员工/售后师傅/工厂应聘者/外包人员指向美的体系的欠薪/劳动纠纷/用工争议
10. **V5.0 重点集团信息**：高管/创始人/投资并购/年报下降/机构唱空/战略质疑/ESG点名/竞品攻击/黑稿传播
11. **品牌/产业链/领导人明确不利描述**：文章中只要有明确不利于美的品牌、美的集团、美的相关业务/产业链、关联公司、核心领导人或高管声誉的描述，都应按负面/敏感进入报送判定。前提是负面主体必须是美的品牌或美的相关产业链/业务/领导人；若负面落在竞品、行业、第三方或宏观环境，美的只是被顺带提及，则不报。

**明确不利于美的的描述包括但不限于**：
- 直接贬损美的品牌、产品、服务、渠道、售后、经营战略、资本市场表现或转型能力。
- 质疑美的相关产业链布局、储能/新能源/医疗/机器人/楼宇科技/工业技术等业务能力、盈利能力、技术短板、亏损、市场份额落后。
- 指向美的核心领导人/高管/创始人的离职、薪酬、名誉、公开回应、战略连续性、资本动作、争议传闻。
- 将美的描述为行业末端、落后方、亏损方、被质疑方、被投诉方、被监管/诉讼/舆论攻击方。

#### 不报送条件（满足任一则不报）

1. **负面不在美的**：原文负面主体是竞品/行业/第三方/城市建设/装修方/平台/个人，美的只是参照/背景
2. **美的是正向参照**：比较关系中美的更强/更好/领先/标尺
3. **美的是历史/行业背景**：历史案例/行业地位/收购背景/供应合作/技术成果/品牌露出/展会露出/榜单参照
4. **内容中性或正向**：导购/参数/教程/招商/广告/好评/产品介绍/获批/合作进展/正向经营成果
5. **普通竞对分析/正常横评**："都不错/各有优势/按需选择/差距不大"
6. **标题党/痛点铺垫**：开头焦虑/避雷/噪音吸引注意，但整体在介绍美的优势
7. **主体无法确认**：文本过短/OCR噪声/链接缺失/内容混乱
8. **二级市场普通行情**：仅为股票日常涨跌、盘口波动、K线/技术指标、成交额、市值变化、资金流入流出、买卖点提示、短线交易情绪，未延伸到美的经营、战略、投资、业务或品牌声誉层面的内容
9. **重复采集无新风险**：同一链接/投诉/视频/传播链/状态流转已报送

#### 弱显性风险规则

即使没有典型负面词，满足以下三项也应报送：
1. 美的是明确涉事主体
2. 描述了异常状态/体验失败/权益受损/质量疑问/服务阻塞/承诺未达成/公众质疑
3. 普通客户会认为这对美的不利

#### 反误报验证（准备报之前必答）

1. 不利含义是否明确落到美的？
2. 美的是否不是地址/背景/历史案例/展会露出/正向参照/普通形容词？
3. 竞品/行业内容中美的是否确实被负面评价？
4. 集团/资本内容是否存在当前风险而非单纯正向进展？
5. AI初始标签提示的风险，标题/摘要/正文能否独立支持？

**任一否 → 不能报送 → 改不报送或人工复核**

#### 反漏报验证（准备不报之前必答）

1. 是否存在美的为涉事主体的具体异常或权益争议？
2. 是否存在弱表达（疑问句/短文本/截图/转发）但在说美的问题？
3. 是否涉及高管/资本/监管/投诉/公共安全/召回/群体影响等敏感事项？
4. 是否因链接短链/转载/截断/正文缺失导致主体/风险未识别？

**任一是 → 不能直接剔除 → 应报送或人工复核**

#### 最终可解释性检查

- 报送理由：`报送，因为{美的主体}被指{具体负面事实}，可能影响{产品口碑/售后评价/集团声誉/资本市场/高管名誉}`
- 不报送理由：`不报送，因为原文主要讨论{真实主体}，美的仅为{背景/列表/地址/正向对比}，负面未落到美的`
- 写不出 → 输出"需人工复核"

### 第4步：分类判定

#### 服务售后 vs 产品售后 vs 产品质量 三分法

| 分类 | 核心定义 | 典型场景 |
|------|----------|----------|
| **服务售后** | 安装/客服/维修态度/乱收费/拖延/失联/商家拒绝退换货 | 空调安装拖延、客服不回复/推诿、安装师傅乱收费、**明确拒绝退货退款** |
| **产品售后** | 维修/换机/保修/申请退货退款（尚未明确拒绝） | 维修无果/三次未修好、保修期内收费、申请退换货流程 |
| **产品质量** | 产品自身异常/缺陷/安全问题 | 漏水/异响/不开机/不制冷/爆裂/漏电/冒烟/爆炸/自燃 |

**优先看投诉核心**：
- 用户主要抱怨客服/退换被拒/安装收费 → **服务售后**
- 用户主要说产品坏/漏/异响/起火/爆炸 → **产品质量**
- 用户主要说维修多次/保修收费/换机申请 → **产品售后**
- 输出前必须做旧标签纠错：凡系统标签、AI初判或正文中出现旧写法 `售后服务`，最终报送标签一律改为客户标签池中的 `服务售后`；不得输出 `【售后服务】`。

#### 易误判标签硬规则

0. **中国区域标签定义**
   - `中国区域` 不是“国内舆情”的泛称，而是 Word 标签规则中的四级业务线：明确属于 C 端/中国区相关，但无法分清具体业务线，或不包含在既有产品类别表内时使用。
   - 若标题、摘要或 `news_content` 已能确认具体产品线，如家用空调、冰箱、洗衣机、小家电、厨热、微烤、全屋智能、高端品牌、年轻化性价比品牌、洗护品牌等，不得仅因发生在国内就改挂 `中国区域`。
   - 仅当多个产品被同一问题一起投诉、没有提及具体产品、或风险主体落在整体客服/售后/门店/渠道/服务体系时，才优先使用 `【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【服务售后】`。

0.1 **品牌层面 vs 产品层面强制分流**
   - 判断标签层级时先看“不利点落在哪里”，不能只看文本里是否出现“品牌”“美的整体”“口碑”“声誉”等词。
   - **产品层面**：凡不利点落在美的产品、家电、白电、单品、品类、耐用性、质量、价格、性价比、使用体验、售后体验、客服体验、安装维修、长期使用表现、与竞品产品对比等，均先归 C端/具体产品线，不得归集团层面。
   - **具体产品线可识别**：如空调、冰箱、洗衣机、小家电、厨热、微烤、全屋智能、COLMO、小天鹅、华凌、东芝白电等，输出 `【{等级}】【C端产品风险舆情】【智能家居事业群】【{具体业务线}】【{舆情分类}】`。
   - **只有泛产品表达、无法识别具体产品线**：如“美的产品”“美的家电”“美的白电”“买美的的”“美的不耐用”等，输出 `【{等级}】【C端产品风险舆情】【智能家居事业群】【中国区域】【{舆情分类}】`。
   - **品牌/集团层面**：只有讨论年度经营、财报、营收利润、资本市场、战略方向、投资并购、集团治理、品牌整体声誉、企业级竞争格局，且不利点没有落到具体产品或泛产品体验时，才输出 `【{等级}】【集团层面】【品牌整体】【{舆情分类}】`。
   - 若同一信息同时包含产品层面和集团层面，优先按更具体、客户可处置的产品/业务线归属；除非集团层面风险明显高于产品问题（如财报暴雷、监管处罚、高管争议）。

1. **客服/售后处理问题 + 明确指向美的品牌的归属边界**
   - 当内容同时满足以下两个条件时，先确认是否能归属到具体产品线：
     1. 明确涉及客服、官方客服、售后、网点、门店、安装师傅、维修人员、京东/天猫/平台客服协同处理、退换货处理、价保处理、补贴处理、服务态度、推诿、拖延、不处理、拒绝受理等“需要客服/售后处理”的问题。
     2. 负面事实明确指向美的品牌、美的官方渠道、美的客服官方微博、美的京东自营旗舰店、美的售后体系或美的相关子品牌官方渠道。
   - 若同一条内容中出现多个产品被投诉，或未提及具体产品、只指向美的整体客服/售后/门店/渠道/服务体系，则归中国区域服务售后，固定优先标签：
     `【{等级}】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【服务售后】`
   - 若内容明确提及某个具体产品或单一产品线，例如小家电、冰箱、洗衣机、家用空调、厨热、微烤等，仍按原产品线归属：
     `【{等级}】【C端产品风险舆情】【智能家居事业群】【{具体业务线}】【服务售后/产品售后/产品质量/营销推广】`
   - 若同一内容既有具体产品故障又有客服/售后处理问题，先按具体产品线归属；再根据用户核心诉求选择末级分类：
     - 核心是客服不处理、处理不当、拒绝价保/退款/换机/维修、服务态度差 → `服务售后`
     - 核心是维修多次、保修收费、换机申请、退货退款流程 → `产品售后`
     - 核心是产品坏、漏、异响、不制冷、安全隐患、质量缺陷 → `产品质量`
   - 只有无法归入具体产品或涉及多个产品/整体服务体系时，才使用中国区域服务售后。

2. **价格权益/价保/补贴争议 + 客服处理不当**
   - 若争议核心是价保、价格保护、补贴、活动承诺、优惠券、差价、价格承诺未兑现，通常归营销推广。
   - 但如果正文重点是客服不告知、拒绝处理、诱导取消订单、推诿、不予受理等处理问题，且明确有具体产品，则仍按具体产品线 + 服务售后或营销推广；只有没有具体产品或涉及多产品/整体服务体系时，才归中国区域 + 服务售后。
   - 标签冲突时，以客户最新标签明细为准；在未给出明细前，优先保留“问题核心”并在判定原因中说明。

3. **领导人/高管舆情固定标签**
   - 只要内容涉及美的核心领导人、创始人或高管的任免、离职、薪酬、名誉、公开回应、争议传闻、资本动作、战略连续性等客户关注事项，使用固定高管标签，不随意套产品或区域标签。
   - 固定5段报送标签：
     `【{等级}】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】`
   - 如触发高敏，在最前方加 `【高敏❗】`：
     `【高敏❗】【{等级}】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】`
   - 纯荣誉榜单、无争议正向露出、普通名单提及，不报送；但一旦涉及薪资、离职、战略担忧、公众质疑、争议回应，按高管舆情报送。

4. **美美家园固定归属**
   - `美美家园` 是美的相关产业线/服务品牌。凡负面或敏感事实明确涉及美美家园、其门店、服务、安装、维修、交付、客服、订单履约、售后处理等，归中国区域服务售后：
     `【{等级}】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【服务售后】`
   - 若同一内容同时出现具体产品，但风险主体主要落在美美家园的服务链路、交付链路或售后链路，仍优先归中国区域服务售后。
   - 仅为美美家园广告、正常介绍、正向导购、无负面事实的品牌露出，不报送。

### 第5步：等级判定（就高原则）

#### 重大危机

- 单篇微博转评赞>5000 / 微信阅读10W+ / 论坛评论>3000 / 小红书转评赞>1000 / 抖音转评赞>10000 / 视频号转评赞>1000
- 成为热搜/微博标签/极可能上升热搜
- 党报央媒/头部财经/境外媒体集中负面
- 省级以上政府介入/市监局处罚/约谈调查
- 2人及以上死亡 / 100名以上投诉受影响
- 严重安全事故/重大法律诉讼/政治敏感/群体健康危害
- 单起损失超300万元
- 跨平台大规模扩散/专项传播链
- 触发国内外差异/社会公平/公共安全/人身伤害/群体权益等高敏场景且明显传播

#### 危机

- 微博转评赞500-5000 / 微信阅读1000-10W / 小红书200-1000 / 抖音2000-10000
- 多家区域/省级/全国媒体报道
- 县地级政府警告/检查/整改/消协问询/市监局立案
- 1人死亡或重伤 / 11-100人受影响
- 核心管理人员/部门涉嫌违法违规
- 损失100-300万元
- 权威媒体持续跟进同一事件
- 集团关联资产持续亏损/甩卖/强负面/投资者集中质疑

#### 异常事件

- 普通用户/自媒体/投诉平台发布负面，暂未大规模传播
- 转评赞<500(微博) / <200(小红书) / <2000(抖音) / <1000(微信)
- ≤10名消费者/员工受影响
- 县级问询/站点转办/一般EHS风险
- 损失10-100万元
- 低互动吐槽（口语化/短视频/图文笔记）
- 竞品对比明确不利但暂无高传播

#### 一般负面

轻微抱怨/无事实细节/传播极低/风险极低。模板不使用时可归入异常事件。

#### V5.0 高敏判定（独立于等级）

**必定高敏**：爆炸、火灾、冒烟、死亡、伤亡、触电、中毒、自燃
**原则高敏**：315、央视、人民日报、新华社、市场监管处罚、监管约谈

- 高敏在完整标签最前方加 `【高敏❗】`
- 高敏格式是固定字符：必须完整输出 `【高敏❗】`。`❗` 不能省略，不能替换成 `!`、空格加符号或其他变体，也不能写成 `【高敏】`、`【高敏!】`、`【高敏 ❗】`、`【高敏❗️】`。
- 高敏不因单条互动低而降级
- 爆炸/火灾/自燃/中毒/触电/重大伤亡 → 通常重大危机
- 老人儿童弱势群体/法律民生节目跟进 → 至少危机

### 第6步：完整标签生成（报送格式）

#### ⭐⭐⭐ 最重要：按客户报送格式输出 ⭐⭐⭐

根据第1-5步的判定结果，按以下规则组装**报送格式标签**：

##### 场景1：高管/创始人相关信息（规则A）

**适用条件**：涉及何享健/方洪波/王建国/顾炎民/管金伟等高管/创始人的任免/离职/薪酬/套现/减持/回应争议/公开表态/名誉争议/传闻/诉讼/监管/负面联想

【{等级}】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】
```

高管/创始人相关信息必须使用上述固定五段标签，不允许输出为 `【{等级}】【集团层面】【品牌整体】【创始人及高管舆情】` 或 `【{等级}】【集团层面】【品牌整体】【高管名誉】`。

**示例**：
```
推送标题：【异常事件】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】
推送标题：【危机】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】
推送标题：【高敏❗】【危机】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】

**输出格式**：
```
【{等级}】【集团层面】【品牌整体】【{对应舆情分类}】
```

**示例**：
```
推送标题：【异常事件】【集团层面】【品牌整体】【经营战略】
推送标题：【危机】【集团层面】【品牌整体】【资本运作】
推送标题：【异常事件】【集团层面】【品牌整体】【合规诉讼】
推送标题：【重大危机】【集团层面】【品牌整体】【环保安全】
```

##### 场景3：C端产品风险（规则C）

**适用条件**：智能家居事业群旗下具体产品的质量问题/售后/服务等负面

**输出格式**：
```
【{等级}】【C端产品风险舆情】【智能家居事业群】【{具体业务线}】【{舆情分类}】

**示例**：
```
推送标题：【异常事件】【C端产品风险舆情】【智能家居事业群】【家用空调】【产品质量】
推送标题：【危机】【C端产品风险舆情】【智能家居事业群】【冰箱】【产品售后】
推送标题：【异常事件】【C端产品风险舆情】【智能家居事业群】【洗衣机】【服务售后】
```

##### 场景4：中国区域营销与服务（规则C）

**适用条件**：无法归入具体产品线的区域服务/售后/营销/活动/门店/会员运营类

**输出格式**：
```
【{等级}】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【{舆情分类}】
```

**示例**：
```
推送标题：【异常事件】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【服务售后】
推送标题：【危机】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【营销推广】
```

##### 场景5：B端业务风险（规则C）

**适用条件**：工业技术/楼宇科技/机器人与自动化/安得/美云/万东/其他ToB业务

**输出格式**：
```
【{等级}】【B端业务风险舆情】【{对应事业部}】【{对应业务线}】【{舆情分类}】
```

**示例**：
```
推送标题：【异常事件】【B端业务风险舆情】【工业技术事业部】【暖通家电部件】【产业生态】
推送标题：【危机】【B端业务风险舆情】【楼宇科技事业部】【暖通】【服务质量】
推送标题：【异常事件】【B端业务风险舆情】【机器人与自动化事业部】【KUKA】【交付争议】
推送标题：【异常事件】【B端业务风险舆情】【其他业务】【安得智联】【产业生态】
推送标题：【异常事件】【B端业务风险舆情】【其他业务】【安得智联】【资本运作】
推送标题：【异常事件】【B端业务风险舆情】【其他业务】【美云智数】【经营战略】
```

**B端标签不得自造**：
- B端输出必须严格使用客户标签池，不得把公司名临时提升为事业部，不得把业务描述临时写成业务线。
- 安得智联固定层级：`【{等级}】【B端业务风险舆情】【其他业务】【安得智联】【{舆情分类}】`；不得输出 `【安得智联】【物流/供应链】`，因为没有“物流/供应链”这个业务线标签。
- 美云智数固定层级：`【{等级}】【B端业务风险舆情】【其他业务】【美云智数】【{舆情分类}】`；不得把美云智数单独作为事业部。
- 安得智联/美云智数的末级分类按内容判断：
  - IPO/递表/上市、估值、融资、股权、关联交易、客户依赖、毛利率/利润率、盈利能力、资本市场质疑 → `资本运作`
  - 业务模式、战略定位、科技属性、出海能力、跨行业复制、增长质量、经营独立性 → `经营战略`
  - 供应链上下游、生态合作、项目交付、客户结构、产业协同，且未明显指向资本市场或战略经营质疑 → `产业生态`
  - 收并购、资产整合、投资标的、战略投资 → `投资并购`

##### 场景6：子品牌风险（规则C）

**适用条件**：COLMO/小天鹅/华凌/东芝白电、厨热、微烤等美的体系内东芝产品的特定负面（非具体产品功能问题而是品牌层面）。东芝电视/电视机/TV不属于美的，不适用本规则。

**输出格式**：
```
【{等级}】【子品牌风险舆情】【智能家居事业群】【{高端品牌/洗护品牌/年轻化性价比品牌}】【品牌形象】
```

##### 场景7：高敏专项（叠加）

任何场景触发高敏时，在标签最前方加 `【高敏❗】`

```
【高敏❗】【{等级}】...后续同上...
```

输出前必须校验：若模型生成了 `【高敏】`、`【高敏!】`、`【高敏 ❗】`、`【高敏❗️】` 或漏掉高敏前缀，统一纠正为 `【高敏❗】`。

```
是否涉及高管/创始人？
├─ 是 → 规则A：【{等级}】【集团层面】【品牌整体】【创始人及高管舆情】【高管名誉】
│
└─ 否 → 是否属于集团层面？
         ├─ 是 → 规则B：【{等级}】【集团层面】【品牌整体】【{对应舆情分类}】
         │
         └─ 否 → 是否属于C端产品？

                  ├─ 是（区域服务）→ 规则C-区域：【{等级}】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【{分类}】

                           └─ 否 → 子品牌/其他（按实际情况归类）

### 第7步：摘要重写

#### 要求

- 让客户**不打开链接也能理解事件核心**：针对美的什么对象、什么问题、诉求/争议点、风险点；不要重复平台、发布人、用户、作者、发布时间等已在独立字段展示的信息
- 摘要的目标是“客户一眼读懂、基本不用再点开原文”：必须说清楚具体对象、具体问题、关键诉求/后果/进展，不能只写“存在争议/引发关注/体验不佳”等泛化描述
- **摘要尽量精简至50字左右，最长不得超过100字**
- 原始摘要在50字以内且事实完整时，可直接保留原文；仍需删除投诉编号前缀等格式噪声（见平台特殊规则）
- 原始摘要超过50字时，优先从原文标题/摘要/正文中挑选关键句子或关键事实片段进行精简，不改写、不添加、不编造
- 超过100字的摘要必须压缩；不得因高管、集团或高敏信息超过100字，只能保留最关键事实
- 凸显美的相关内容，不放大量行业背景/竞品参数/无关叙事
- 从 `news_content` 补齐问题对象/事实/诉求/风险落点
- 摘要不能跟AI总结一模一样

#### 公式

```
{美的主体/品牌/具体产品/服务链路} + {核心问题/争议事实/不利描述} + {诉求/金额/进展/风险点}
```

摘要直接写核心事件，不写采集叙述、平台叙述或用户叙述。不得以“某某用户在某某平台发帖称/投诉称/发布称/表示/反映/吐槽”等开头；平台、发布人、发布时间、发布ID、用户/作者身份已经在独立字段展示，摘要中不重复。

摘要输出前自检：
1. 客户不点链接，能否知道具体说的是哪个美的对象/产品/业务？
2. 能否知道具体问题是什么，而不是只看到“负面/争议/投诉/体验差”？
3. 若有诉求、金额、处理结果、风险点，是否保留了最关键的一项？
4. 是否去掉了平台、用户、作者、发布时间、编号等重复字段？
任一答案为“否”，必须重写摘要。

示例：
- 错误：`黑猫投诉用户勇敢喵发帖称，其在京东购买美的净水器后遇到价保问题。`
- 正确：`美的京东自营旗舰店净水器被投诉保价承诺未兑现，用户要求补差价。`
- 错误：`抖音用户发布视频反映美的天猫店冰箱存在价差问题。`
- 正确：`美的天猫店冰箱存在价差争议，用户要求退还差价。`

#### 禁止

- 机械复制原始摘要乱码/OCR噪声/话题标签/表情堆叠
- 摘要前置采集话术：如“某某于黑猫投诉平台发起”“某用户在小红书发帖称”“今日头条用户发布短视频称”“发布者在抖音表示”“该网友反映”“作者发布图文/视频称”等；这些信息属于发布平台/发布人字段，不进摘要
- 摘要中重复平台、作者、用户、网友、发布时间、发布ID、投诉编号等元数据；除非这些信息本身构成风险事实
- 把"可能/疑似"写成确定事实
- 将"已回复/已完成"误写为解决（除非原文明确显示接受）

#### 摘要精简必须保留

1. 具体产品名称：如养生壶、豆浆机、洗碗机、护眼灯、冰箱、空调、洗衣机、热水器、蒸烤箱等；不能泛化成"商品"或"产品"。
2. 产品标识或平台标识：如 `@美的空调`、`#华凌空调`、`#小天鹅洗衣机` 等，若这些标识帮助确认主体，应保留。
3. 品牌名：美的、小天鹅、华凌、东芝白电、COLMO等不能省略；东芝电视不属于美的，不能因出现“东芝”就纳入报送。
4. 原文摘要未提具体产品但AI总结或正文能确认产品名称时，可从AI总结或 `news_content` 中提取产品名称补充进摘要；补充必须有原文或系统字段依据。
5. 同一信息在不同平台发布时，打标、AI总结、AI研判原则上保持一致；平台、作者、链接、互动数据等差异只体现在对应字段或重复/传播链状态中，不写进摘要、AI总结、AI研判正文。

### 第7.5步：AI总结/AI研判修正规则

AI总结和AI研判是报送给客户的重要字段，必须二次修正，不能直接沿用系统初版。

#### AI总结规则

1. **禁止编造**：原文标题、摘要、正文、页面字段或系统可信字段中没有的内容，不能出现在AI总结中。
2. **忠实原文**：AI总结必须准确反映原文核心诉求、事实或观点；不能把中性观点写成投诉，不能把疑似写成确定事实。
3. **产品/品牌确认**：产品或品牌信息只能从标题、摘要、`news_content`、AI总结、AI研判等正文语义字段中提取；系统标签中的产品/品牌可能错误，不可直接采用。
4. **精简原则**：AI总结尽可能一句话概括核心事实，但具体产品名称和品牌名不能省略。
5. **不与摘要完全相同**：AI总结应是事实概括，摘要是原文关键句精简，两者不能一模一样。
6. **发布平台与内容来源分离**：AI总结必须尊重实际发布平台/链接，但不重复写平台名、发布人、作者、网友、用户等元数据。若链接是抖音/小红书/今日头条等社交平台，但内容转述、截图或搬运了消费保/黑猫/12315投诉，只能写“内容涉及/转述/展示消费保投诉”，不能写成“消费者在消费保平台投诉”作为当前发布事实。
7. **禁止平台话术**：AI总结不得以“某平台发布/某平台用户发帖/某网友称/作者表示/发布者反映”等开头，也不要写“搜狐网刊发”“今日头条用户发帖”“抖音内容展示”等平台主语；直接写事件事实。

#### AI研判规则

1. **视角限制**：AI研判必须站在美的客户视角评估风险、传播和潜在影响；最终读者是美的客户，不是内部执行人员。
2. **不编造处置动作**：原文未出现监管介入、品牌回应、消费者诉求升级、已解决等信息时，研判中不得写成已发生。
3. **精简原则**：AI研判尽可能一句话，只说明对美的的潜在影响、传播态势、风险性质或升级可能。
4. **低互动表达**：低互动不能作为不报送理由，只能写"当前传播有限"。
5. **重复表达**：同一信息跨平台发布时，AI研判口径保持一致；不同平台、作者、链接、互动数据等差异只体现在对应字段或重复/传播链状态中，不在AI研判中重复平台名。
6. **禁止内部推送话术**：AI研判中不得出现 `建议报送`、`建议推送`、`推送` 等内部报送流程用语。
7. **处置类建议酌情使用**：`建议关注`、`建议核实`、`建议处理`、`建议跟进` 可在确有必要时使用，但不能机械添加；优先写风险判断和传播态势。不得给竞品、平台或第三方品牌提建议。
8. **客户交付表达**：可写成 `该内容可能影响美的{产品/服务/品牌/高管/资本市场}口碑，当前传播有限/存在扩散风险/涉及高敏议题。` 如需补充可写 `建议关注后续进展/建议核实相关事实`。
9. **低信息量内容也必须研判**：只要结论为报送，必须输出 AI总结和 AI研判，不能空字段。若原文只有一句情绪吐槽或事实不足，不得编造产品、原因、诉求或风险路径；应写成 `内容表达对美的不满，未说明具体产品或事件。` / `该内容为低信息量负面吐槽，具体风险点不明，但存在美的品牌负面情绪。`
10. **禁止平台主语**：AI研判不得以“某平台发布/某平台用户发帖/某媒体刊发/该网友”等平台或用户主语开头；平台、发布人和作者已经在字段中展示。研判直接写风险性质、传播态势和对美的的潜在影响。

#### 字段格式规则

- 所有字段值不得以中文冒号 `：` 或英文冒号 `:` 开头。
- 字段内容不能出现多余的前置冒号、空冒号或重复字段名。
- 若字段无内容，按模板写 `无` 或 `-`，不要留空导致格式错位。
- 报送字段必须严格按照样例模板逐行输出，不得因为字段值为 `无`、`0`、`-` 或空值而删除整行。
- `IP属地` 是固定必填行：有明确属地写属地；没有、不可见、未采集时必须写 `IP属地：无`，不得省略该行。

## 第四部分：判定规则原则（核心方法论）

### 只能写判定规则，不能把关键词当规则

任何词/平台/产品名/AI标签都只能作为**线索**；最终结论必须来自**语义判断**。

### 标题+摘要+正文联合理解（三者缺一不可）

- 标题 → 话题入口和潜在风险
- 摘要 → 核心事实落点
- 正文(news_content) → 补齐截断/OCR缺失/转发上下文/真实立场
- 三者方向不一致 → 以共同呈现出的**完整语义结论**为准

### 四层分析框架

| 层次 | 作用 | 不能做什么 |
|------|------|-----------|
| **召回层** | 解释系统为什么采集到这条 | 不能当作报送依据 |
| **主体确认层** | 判断涉事主体是否属美的体系 | 不能跳过语义确认 |
| **排除验证层** | 判断召回原因是否普通语言/地址/背景 | 不能机械剔除有风险的 |
| **风险判定层** | 判断是否存在落到美的的风险 | 唯一决定报送的层次 |

**只有主体确认层 + 风险判定层都成立 → 才能报送**

### 2026年6月校准口径

#### 1. 只报"内容语义上的负面/敏感/不利"

- 系统情感正/中性但实质负面（真实投诉/价保拒/售后推/产品坏/高管薪酬争议）→ **报**
- 系统情感负但负面不在美的（竞品翻车/美的是优势方/行业背景）→ **不报**
- 标题看似负面但正文在夸美的 → **不报**
- 标题/摘要/正文出现焦虑/避雷/噪音等词但实际在做导购铺垫 → **不报**

#### 2. 正向导购/种草/广告排除

产品种草/晒单/开箱/购买攻略/榜单推荐/参数科普/购买成功/上车成功/价保成功/售后无忧/生活痛点推荐美的解决方案/正向安全表述/抬高美的 → 即使有风险线索词也不报。除非进一步得出"美的产品质量差/售后差/体验差"等不利结论。

#### 3. 普通竞对分析和中性横评排除

"都不错/各有优势/各有侧重/都比较成熟/按需选择/价格差不多求建议/正常横评/选购指南/行业列表/财经横向描述" → 不报。"美的不如/弱于/没有可比性/质量差/售后差/技术落后" → 报。

**竞对关系标签使用边界**：
- `竞对关系` 是五级舆情分类，用于明确出现竞争关系风险，且不利判断落在美的或美的相关业务上。
- 适用场景包括：竞品被明确描述为压制/赶超/替代美的，美的被指在技术、质量、价格、渠道、服务、市场份额、战略布局、资本表现等方面落后或受冲击；竞品营销、舆论攻击、行业排名、市场份额变化直接形成对美的不利影响。
- 不适用场景：普通选购指南、正常参数横评、行业名单罗列、"格力美的都不错"、抬高美的的正向对比、无明显负面落点的竞争格局分析。
- 标签归属按风险落点选择，不得只因影响“品牌口碑/品牌声誉”就归集团层面：
  - 具体产品线竞争风险：如“美的空调不如格力”“美的冰箱不耐用”“小天鹅洗衣机不如竞品”，用 `【{等级}】【C端产品风险舆情】【智能家居事业群】【{具体业务线}】【竞对关系】`。
  - 泛产品竞争风险：如“买美的的只能短期用几年”“美的产品便宜但不耐用”“美的家电品质不如格力”，未说明具体品类时，用 `【{等级}】【C端产品风险舆情】【智能家居事业群】【中国区域】【竞对关系】`。
  - 集团/品牌战略竞争风险：如年度经营状况、财报表现、战略方向、资本市场、企业级竞争格局、集团品牌整体被竞品压制等，且不利点没有落到产品使用/质量/耐用/售后体验，用 `【{等级}】【集团层面】【品牌整体】【竞对关系】`。
  - B端业务竞争风险：如楼宇科技、机器人、能源、医疗、安得智联等业务被竞品压制或替代，用 `【{等级}】【B端业务风险舆情】【{事业部}】【{业务线}】【竞对关系】`。

#### 4. 正常背景/合作/技术成果排除

美的是客户/合作伙伴/参展企业/技术案例/品牌墙/门店背景/地图定位/多个企业之一 → 不报，除非同时指向美的当前负面风险。

#### 5. 普通语言误召回排除

完美/甜美/精美/优美/爱美/物美价廉/地名/店名/截图背景/无关OCR中出现"美的" → 不报。

#### 6. 公开投诉重点保留

黑猫/消费保/12315/问政/民生帮办中投诉对象/商品事实指向美的 → 报。投诉量统计/解决率榜单抬高美的 → 不报。同一投诉状态流转无新事实 → 不重复报。

#### 7. 产品/服务消费者风险

产品坏了/过保就坏/主板压缩机等故障/不制冷漏水漏电异响冒烟爆炸/安装不规范未抽真空/物流延迟/客服不处理售后推诿/价保补贴未兑现 → 报。小红书/抖音低互动只要事实明确指向美的也报。

#### 8. 高管/核心领导重点报

何享健/方洪波/卫昶/赵磊等的任免离职薪酬套现减持回应争议名誉传闻诉讼监管 → **重点报**。高管薪资/离职/CTO变动/战略连续性担忧 → 即使偏中性也报。纯荣誉名单/企业家榜单/无争议露出 → 不报。

#### 9. 高敏安全/合规

真实爆炸火灾冒烟死亡伤亡触电中毒自燃 + 风险落在美的 → **报+加高敏**。"不用担心中毒/防爆锅/安全稳定/0故障"等正向 → 不触发高敏。

#### 10. B端/集团/资本风险

楼宇科技/多联机/工程故障/库卡/安得/美云/万东/科陆/美的能源出现亏损/诉讼/监管/资本质疑/项目交付/客户投诉/战略争议 → 报。技术突破/订单增长/获批资质/产业链自主/出海报道 → 不报（除非同时亏损/诉讼/监管）。

**二级市场信息分流**：
- 不报：仅来自股票行情/二级市场数据的普通日常涨跌、盘中波动、技术面分析、短线买卖点、成交量/成交额/资金流向/市值变化、股民无实质事实的普通情绪表达，且未涉及美的品牌声誉、经营问题、战略方向、投资并购、业务范围、财报压力或资本承诺。
- 报：股票或投资者讨论延伸到美的经营质量、利润/营收压力、财报异常、战略投资、业务转型、资本运作、分红承诺、并购整合、关联资产亏损、管理层决策、品牌声誉、行业竞争地位等舆论判断。
- 标签：经营/战略/财报压力一般用 `【{等级}】【集团层面】【品牌整体】【经营战略】`；投资并购用 `投资并购`；资本动作、股东利益、分红、融资、股价异常与资本承诺用 `资本运作`；竞品导致资本市场或经营竞争压力用 `竞对关系`。

### 排除词的作用

排除词 = "这可能不是美的风险"的**提示信号**，但必须经过语义确认：

- 美的只是语言片段/地理定位/图片背景/展会露出/品牌墙/多个企业之一/历史案例/技术成果/供应合作/竞品对照标尺 → 且没有当前不利事实 → 不报

- 导购/榜单/广告/招商/交易/教程/参数比较但进一步形成对美的不利判断 → **报**或人工复核
- 排除场景与风险场景同时出现 → **以风险落点优先**

## 第七部分：质量要求

1. **不能只依赖 AI 初始标签** → 必须结合标题/摘要/正文/链接/投诉对象和主体相关性
2. **不能依赖召回词/平台/AI标签/固定表达作结论** → 这些只能作证据线索
3. **每条结论必须完成"主体-事件-负面落点-客户风险"语义链路** → 并写出可解释理由
4. **大批量处理时保留判定原因字段** → 建议"主体类型/负面落点/反误报验证/反漏报验证/复核建议"
5. **摘要必须二次提炼** → 尽量精简至50字左右且不得超过100字；从news_content提取关键事实，突出美的主体和核心问题
6. **低互动负面仍可报送** → 互动量只影响等级和研判，不决定报不报
7. **高管类优先级高** → 变形词/职务称谓/关联事件都要识别
8. **转发/评论/表情包负面指向美的也要识别**
9. **重复采集必须提示** → 不让人工反复打开同一投诉/链接
10. **输出标签必须使用客户标签池** → 不再输出旧的271条精细标签
10.0 **禁止虚拟/创造标签** → 报送标签的每一段必须来自客户标签池或本文固定映射；不得根据语义自行创造事业部、业务线或舆情分类。若语义判断与标签池不完全匹配，优先使用客户已给定的最近固定标签，并把细节写入摘要/AI总结/AI研判。
10.1 **高敏前缀必须固定** → 触发高敏时只能输出 `【高敏❗】`，不得写成 `【高敏】`、`【高敏!】` 或漏掉 `❗`
11. **处理后必须抽查三组样本** → 已报送/未报送/人工实报但本轮判否 → 用结果继续修正
12. **明确不利于美的品牌/产业链/领导人的描述必须按负面处理** → 但必须确认负面主体是美的品牌、美的相关业务/产业链或核心领导人
13. **AI总结禁止编造** → 只能写原文或可信字段中存在的事实，不能补充不存在的诉求、监管、回应或结果
14. **AI研判必须站在美的客户视角** → 评估对美的的风险、传播态势和潜在影响；不得写建议报送、建议推送、推送等内部流程话术；建议关注/核实/处理/跟进可按内容酌情使用
14.1 **报送必须有AI总结和AI研判** → 低信息量吐槽也要输出克制字段；不得空字段，不得编造具体产品、原因、诉求或风险路径
15. **产品/品牌不能直接取系统标签** → 必须从标题、摘要、news_content、AI总结、AI研判等语义字段确认
16. **摘要要让客户一眼读懂且基本不用点原文** → 尽量50字左右、最长100字；必须包含具体对象、具体问题、关键诉求/后果/进展；不写泛泛的“存在争议/体验不佳”；不重复平台、发布人、用户、作者、发布时间、发布ID等字段
17. **字段格式不得以冒号开头** → 所有字段值不得以 `：` 或 `:` 开头
18. **小红书需特殊处理** → 链接替换为 xhslink.com 短链，发布ID需提醒人工核验修改
19. **发布ID必须干净** → 只写编号/账号ID本体，不混入平台名、域名、链接、字段名、昵称、括号说明或其他噪声
20. **发布平台与内容来源不能错位** → 链接是抖音/小红书/头条就按当前平台输出；正文提到消费保/黑猫/12315只能作为内容来源，不能覆盖发布平台、发布ID、AI总结主语
21. **投诉平台有编号时发布ID写编号** → 摘要删除投诉编号前缀，AI总结/AI研判不得出现投诉编号，末尾不再输出投诉编号字段
22. **12315必须特殊清洗** → 发布ID只取 `regiNo`/纯登记编号；删除 `】投诉信息`、`投诉问题`、`处理结果` 等拼接噪声；`售后服务` 输出前改为 `服务售后`
23. **IP属地缺失写无且行不能删除** → 不写中国，不猜测；即使为 `无` 也必须输出 `IP属地：无`
