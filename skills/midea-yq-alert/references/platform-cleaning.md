# 平台字段清洗与去重

本文件用于清洗发布平台、发布ID、链接、IP属地、投诉编号和平台噪声，并辅助生成去重键。

## 通用原则

- `发布平台` 必须跟当前链接和采集页面一致，不能被正文中提到的投诉平台、转载来源或截图来源覆盖。
- `发布ID` 必须是编号或账号ID本体，不得混入平台名、域名、链接、字段名、昵称、投诉对象、括号说明或多余符号。
- 作品ID不等于发布ID；社交平台发布ID优先写发布者账号ID。
- IP属地不可见、缺失或未采集时统一写 `无`，不得猜测为中国。
- 投诉编号只用于发布ID、去重键和内部核验，不放入摘要、AI总结、AI研判，也不额外输出“投诉编号”字段。

## 发布ID取值

| 平台类型 | 发布ID取值 |
|---|---|
| 黑猫/消费保/12315/问政等投诉政务平台 | 投诉编号、工单编号、登记编号 |
| 小红书/抖音/快手/今日头条/微博/B站/微信/视频号 | 发布者的平台账号ID |
| 发布人ID为空 | 优先写页面可见作者账号ID，仍无则写发布人名称或 `无` |
| IP属地不可见 | `无` |

## 社交平台转述投诉

当前链接为抖音、小红书、今日头条、微博等社交平台时，即使正文出现“消费保投诉”“黑猫投诉”“12315投诉编号”：

- 发布平台仍写当前社交平台。
- 发布ID写社交平台作者账号ID。
- 正文中的投诉编号不能替代社交平台发布ID。
- 摘要可写“内容涉及/转述/展示某投诉”，但不能以“消费保投诉：”开头。
- AI总结/AI研判直接写事件事实，不用平台或用户作为主语。

推荐表达：

```text
美的天猫店冰箱价差投诉被转述，用户要求退还差价。
```

## 小红书

- 链接必须替换为 `xhslink.com` 短链接格式。
- 系统给出的小红书号或发布ID可能不准确，需要人工或技术侧核验。
- 每次输出小红书报送内容时，必须在客户正式报送内容外额外提醒：

```text
小红书发布ID需手动核验修改，链接需替换为xhslink.com短链接。
```

提醒不要混入客户正式报送正文。

## 黑猫投诉、消费保、12315等投诉平台

有投诉编号时：

1. 发布ID改为投诉编号。
2. 摘要删除“投诉编号xxxx”“【投诉编号：xxxx】”等前缀。
3. 摘要保留投诉对象、问题、诉求、金额、进度等核心事实。
4. AI总结和AI研判不得出现投诉编号。
5. 末尾不再额外输出“投诉编号”字段，也不要输出“投诉编号：无”。

黑猫投诉：

- 只写纯投诉编号，如 `17398633744`。
- 不得写 `投诉编号17398633744`、`黑猫投诉17398633744`、`tousu.sina.com.cn`。

## 12315

12315数据必须额外清洗：

1. 发布ID只取 `regiNo` 或正文中的纯登记编号，例如 `1440606002026042120192653`。
2. 链接中 `regiNo=` 后的值优先于页面拼接文本。
3. 链接参数中的 `keyword=`、企业名称、`pageNumber`、`creditNo` 不得进入发布ID。
4. 若系统把 `】投诉信息：...`、`投诉问题：...`、`处理结果：...` 拼进发布ID，必须截断，只保留编号本体。
5. 摘要删除 `12315平台投诉编号xxxx】`、`投诉信息：`、`投诉问题：`、`处理结果：`、消费者手机号尾号、用户ID等字段噪声。
6. 摘要只保留被投诉主体、购买/服务方式、核心售后或质量问题、诉求、处理结果。
7. 12315原文中的 `售后服务->其他售后服务问题` 是问题描述，不是最终标签；最终标签必须使用 `服务售后`。

若未提及具体产品，仅为其他销售服务或其他售后服务问题，且主体为美的相关公司或渠道，标签使用：

```text
【{等级}】【中国区域营销与服务舆情】【智能家居事业群】【中国区域】【服务售后】
```

若提及具体产品但核心是客服、渠道、安装、服务态度、拖延、拒绝处理等服务售后问题，一级标签仍统一使用 `中国区域营销与服务舆情`，业务线可保留具体产品线：

```text
【{等级}】【中国区域营销与服务舆情】【智能家居事业群】【{具体业务线}】【服务售后】
```

若产品故障已经进入维修、换机、保修、退货退款、售后上门等处理链路，末级分类使用 `产品售后`。

若投诉核心是价保、补贴、活动承诺、优惠券、差价等价格权益或营销争议，按客户最新版 Word 归 `中国区域营销与服务舆情`，末级分类统一使用 `内销业务`。若核心是客服拒绝处理、推诿、安装/物流等服务问题，则使用 `服务售后`。

## 去重键提取

平台ID提取规则：

- 黑猫：`/complaint/view/{编号}`
- 12315：`regiNo=` 或文本中的投诉编号
- 消费保：文本中的“投诉编号”
- 抖音：`/video/{视频ID}`
- 小红书：`/explore/{笔记ID}` 或 `/discovery/item/{笔记ID}`；短链需转换或人工核验
- 微博：微博正文ID、转发链原短链或正文核心文本
- 今日头条：`/item/{ID}` 或 `/w/{ID}`
- 微信文章：`__biz + mid + idx + sn`
- 视频号、快手、B站：视频ID

去重输出应说明：

- 重复类型：同链接、同投诉编号、同内容签名、同传播链。
- 已报送记录：日期、标题、链接或编号。
- 本次变化：无新增事实、新增状态、新增传播、新增诉求。
- 处理建议：合并备注、作为进展更新报送或忽略重复采集。
