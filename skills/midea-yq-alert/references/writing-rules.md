# 摘要、AI总结与AI研判写作规则

本文件用于把原始标题、摘要和正文整理成客户可直接理解的报送内容。

## 摘要

目标：客户不打开链接，也能知道美的什么对象出了什么问题、诉求或风险点是什么。

要求：

- 尽量50字左右，最长100字。
- 直接写核心事件，不写平台叙述、采集叙述或用户叙述。
- 必须突出美的主体、具体产品/业务/服务链路、核心问题、诉求/金额/进展/风险点。
- 不重复发布平台、发布人、用户、作者、发布时间、发布ID、投诉编号等元数据。
- 从 `news_content` 补齐问题对象、事实、诉求和风险落点。
- 原始摘要50字以内且事实完整时可保留，但必须删除投诉编号前缀和字段噪声。
- 超过100字必须压缩。
- 摘要不能与AI总结完全相同。
- 当原文整体只有少量篇幅提及美的负面点时，摘要必须根据正文整体主线如实体现“局部/少量提及”的事实，不得写得像通篇都在批评美的。不要机械使用固定比例阈值；应结合正文篇幅、段落重心、标题导向、论述主线和美的负面点出现位置，判断是否需要使用“原文少量提及”“文中局部提到”“文章在对比/列表中提及”等克制表达，避免放大风险、制造客户舆情焦虑。

推荐公式：

```text
{美的主体/品牌/具体产品/服务链路} + {核心问题/争议事实/不利描述} + {诉求/金额/进展/风险点}
```

低占比提及公式：

```text
原文少量/局部提及{美的主体/产品/业务}存在{负面点/不利对比}，整体内容主要讨论{原文主线}。
```

错误示例：

```text
黑猫投诉用户勇敢喵发帖称，其在京东购买美的净水器后遇到价保问题。
```

正确示例：

```text
美的京东自营旗舰店净水器被投诉保价承诺未兑现，用户要求补差价。
```

## 摘要禁止项

- 不以“某用户在某平台发帖称”“发布者表示”“网友反映”“作者发布视频称”等开头。
- 不机械复制乱码、OCR噪声、话题标签、表情堆叠。
- 不把“可能/疑似”写成确定事实。
- 不将“已回复/已完成”误写为已解决，除非原文明确显示用户接受或问题解决。
- 不泛写“存在争议”“引发关注”“体验不佳”，必须写出具体问题。
- 不把原文中的低占比美的负面提及包装成整篇文章主旨。例如原文主要讨论行业或多品牌对比，只有一两句提到美的不利点时，摘要不得写成“美的被集中质疑/文章批评美的...”。

## 摘要必须保留

- 具体产品名称，如养生壶、豆浆机、洗碗机、护眼灯、冰箱、空调、洗衣机、热水器、蒸烤箱。
- 能确认主体的品牌或标识，如美的、小天鹅、华凌、COLMO、东芝白电。
- 关键诉求、金额、处理结果或安全风险。
- 原文没有具体产品但正文、AI总结或AI研判可信字段能确认产品时，可补充；补充必须有依据。
- 原文负面篇幅占比。若美的负面点只是文章中的低占比内容，摘要必须保留这个比例感，可写“少量提及”“局部提及”“作为对比项提到”。

## 低占比负面提及

当 `news_content` 中美的负面点只占原文较小篇幅，或仅出现在列表、横评、行业分析、转载摘录、评论区单句中时：

1. 仍需判断负面是否明确落到美的；明确落到美的时可以报送。
2. 摘要必须弱化篇幅感，说明这是局部提及，不得让客户误以为通篇围绕美的负面展开。
3. AI研判应写明“原文对美的负面提及占比较低/当前风险点较局部”，再评估是否存在扩散风险。
4. 若美的只是顺带提及且没有明确不利判断，按不报送或人工复核处理。
5. 不要把“5%”或任何固定比例当作硬规则；核心是忠实反映原文整体结构和负面落点权重。

推荐摘要：

```text
原文在多品牌空调对比中局部提到美的柜机能效不及竞品，整体内容主要讨论618空调市场表现。
```

不推荐摘要：

```text
文章批评美的空调能效落后竞品，品牌竞争力不足。
```

## AI总结

AI总结是事实概括，不是风险判断。

规则：

- 忠实原文，不编造原文没有的诉求、监管、回应、结果或处置动作。
- 准确反映核心事实、诉求或观点。
- 不能把中性观点写成投诉，不能把疑似写成确定事实。
- 产品或品牌只能从标题、摘要、正文、AI总结、AI研判等语义字段确认；系统标签不可直接采用。
- 尽量一句话。
- 不与摘要完全相同。
- 不以平台、用户、作者、网友为主语。

低信息量内容可写：

```text
内容表达对美的不满，未说明具体产品或事件。
```

## AI研判

AI研判是客户视角的风险判断。

规则：

- 站在美的客户视角评估风险、传播态势和潜在影响。
- 不编造监管介入、品牌回应、消费者诉求升级、已解决等原文没有的信息。
- 尽量一句话。
- 低互动不能作为不报送理由，只能写“当前传播有限”。
- 不以平台、用户、作者、网友为主语。
- 不出现 `建议报送`、`建议推送`、`推送` 等内部流程话术。
- 可酌情写 `建议关注`、`建议核实`、`建议处理`、`建议跟进`，但不要机械添加。

推荐表达：

```text
该内容可能影响美的{产品/服务/品牌/高管/资本市场}口碑，当前传播有限。
```

高敏表达：

```text
该内容涉及{火灾/爆炸/伤亡/监管等}高敏风险，若事实扩散可能放大美的安全与品牌声誉压力。
```

低信息量内容可写：

```text
该内容为低信息量负面吐槽，具体风险点不明，但存在美的品牌负面情绪。
```

## 字段格式

- 所有字段值不得以中文冒号或英文冒号开头。
- 无内容时写 `无` 或 `-`，不要留空导致格式错位。
- 报送字段必须逐行保留，不得因为字段值为 `无`、`0`、`-` 或空值而删除整行。
- `IP属地` 是固定必填行；缺失写 `IP属地：无`。
